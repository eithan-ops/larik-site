# LARIK Games — משחקי חברה שהטלפון הוא אביזר בהם 🎮

QR → כולם בפנים תוך 5 שניות → המארח בוחר משחק → הסלון משחק.

## מבנה (עמוד השדרה)
```
shared/protocol.ts     ← הפרוטוקול: הודעות, קטלוג, טיפוסים (לקוח+שרת)
server/src/engine.ts   ← מנוע החדרים: שחקנים, פאזות, cues מתוזמנים — עצמאי מטרנספורט
server/src/games/*.ts  ← משחק = מודול: onStart/onMessage/dispose (להוסיף משחק = קובץ+שורה)
server/src/decks.ts    ← חפיסות תוכן (עקרון AI-בזמן-בנייה)
client/src/lib/connection.ts ← סנכרון שעונים ±10-30ms + ביצוע cues
client/src/lib/audio.ts      ← צלילים מסונתזים, מתוזמנים ברמת הדגימה
client/src/lib/sensors.ts    ← חימוש (הרשאות iOS), זיהוי מצח/הצצה בג'ירו
client/src/games/*.tsx       ← מסך המשחק (רישום ב-registry.tsx)
```

## איך התזמון עובד (הדרישה המרכזית)
1. כל לקוח מסנכרן שעון מול השרת (8 פינגים, שומרים את בעל ה-RTT הנמוך) → offset מדויק.
2. אירוע בו-זמני = `cue {at, d}`: השרת קובע זמן-שרת עתידי (עם מרווח ביטחון 350ms),
   וכל לקוח מבצע בזמן המקומי המתאים. סטייה בפועל: <30ms.
3. מדידת תגובה (פודים): הנגיעה נחתמת בזמן-שרת אצל הלקוח — רשת איטית לא מעוותת מדידה.

## הרצה
```bash
cd server && npm install && npm start        # שרת על :8787
cd client && npm install && npm run build    # השרת מגיש את ה-dist
# פיתוח: npm run dev בשתי התיקיות (ויטה מפרוקסי ל-8787)
```
טלפונים אמיתיים דורשים HTTPS (חיישנים). הכי קל לפיילוט בסלון:
```bash
npx cloudflared tunnel --url http://localhost:8787   # כתובת https ציבורית בחינם
```

## בדיקות
```bash
cd server && npm test   # 17 בדיקות: מנוע, שני המשחקים, מתמטיקת הסנכרון
```

## מפת דרכים
- פריסה קבועה: השרת רץ על כל Node host (Fly/Railway/VPS); עתידי — adapter ל-PartyKit/Durable Objects (המנוע לא משתנה).
- עטיפת נייטיב: הלקוח הוא SPA טהור — עטיפה ב-Capacitor מוסיפה רטט iOS, בהירות מלאה, הפעלה ברקע.
- משחק חדש: מודול שרת (games/x.ts) + מסך לקוח (games/x.tsx) + שורה בקטלוג (protocol.ts) + שורה ברישום.
- AI: חפיסות נוצרות בזמן בנייה; "חפיסה אישית" ב-Gemini Flash-Lite (שכבת חינם) — הרחבה מתוכננת.
